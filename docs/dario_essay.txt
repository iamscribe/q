THE DARIO EQUATION — SEVEN SIGNALS

At the center of dario.c sits a probability equation that replaces the
transformer's attention mechanism:

    p(x | Phi, C, V) = softmax(
        (B + alpha_mod * alpha * H_v + beta_mod * beta * F_v
         + gamma_mod * gamma * A + delta * V + sw * S + T)
        / (tau_mod * tau * velocity_temperature)
    )

Where a standard transformer computes softmax(QK^T / sqrt(d)) * V using learned
projections, this equation computes seven physically interpretable forces acting
on a shared vocabulary. No learned projections. No multi-head anything. No
feed-forward layers. Seven terms, each contributing logits from a different
angle, summed, temperature-divided, softmaxed.

The seven signals:

B — SEQUENTIAL CHAIN. Bigram transition probabilities. What word tends to follow
the previous word. The simplest force: local coherence, word-to-word inertia.
Coefficient starts at 8x, boosted to 10.4x in autumn (seasonal modulation) and
during RUN velocity. This is the backbone. Every natural language has it. Every
child learns it first. The chain never disappears — language always needs local
coherence — but the organism learns to share the stage as it matures.

H — HEBBIAN RESONANCE. Co-occurrence memory. A sparse matrix recording which
words appeared near which other words, weighted by distance. Window of plus or
minus 5 tokens at ingestion time. At generation time, the last 8 context tokens
vote on every vocabulary word through their co-occurrence counts, weighted by a
learnable positional profile — 36 Hebbian parameters (32 distance weights plus
4 token class modifiers) that adapt through conversation. Content words gain
about 18 percent weight over function words after just 15 exchanges. Emergent,
not trained.

The theoretical basis is Hebb's rule: neurons that fire together wire together.
PLOS Computational Biology (2024) proved it mathematically — accumulated
Hebbian learning equals unnormalized dot-product attention. Co-occurrence IS
attention. So instead of learning QKV projections through billions of gradient
steps, the organism grows them through conversation.

F — PROPHECY FULFILLMENT. After generating each token, the organism predicts
what comes next (strongest co-occurrence partner). That prediction becomes a
prophecy. If it goes unfulfilled, its debt grows logarithmically with age.
Unfulfilled intentions create generation pressure — a cosine-similarity pull
toward the prophesied token, weighted by prophecy strength and accumulated debt.
This is not beam search. This is a child who started saying something and feels
the need to finish. Max 32 active prophecies. Age limit 50 steps. Fulfilled
prophecies are cleared — debt zeroes, the field exhales.

A — DESTINY ATTRACTION. The exponential moving average of all context
embeddings: destiny[d] = 0.1 * embed[d] + 0.9 * destiny[d]. A 64-dimensional
semantic compass drifting with the dialogue. Cosine similarity between each
vocabulary word's embedding and the destiny vector, scaled by destiny's
magnitude. The organism does not follow topics. It drifts toward them. The field
has mass. Destiny updates apply Rotary Position Embedding (RoPE), encoding when
a word appeared, not just what it was. Position-aware destiny.

V — VISUAL GROUNDING. A parallel perceptual embedding space (different hash
seed, orthogonal to semantic embeddings). Each word gets both a semantic
embedding and a visual prototype. A visual context vector (EMA, like destiny but
in perceptual space) tracks what was "seen." V computes cosine similarity
between each word's visual prototype and accumulated visual context. Visual
co-occurrence enriches H and F multiplicatively: H_v = H + lambda * V * H.
Visual grounding amplifies existing signals rather than creating new ones — a
word must first have semantic relevance before visual grounding strengthens it.

S — SUBWORD STRUCTURE. A placeholder in dario.c (the term exists but contributes
zero energy). In Leo, S is driven by a SubwordField BPE tokenizer running in
parallel with the word-level system — capturing punctuation, morphology, suffix
patterns. S awaits activation.

T — TRAUMA GRAVITY. When trauma exceeds 0.3, the first 50 seed words (the
physics vocabulary: resonance, field, destiny, prophecy, decay) receive
gravitational logit boosts. The boost decreases linearly — word 0 gets full
weight, word 49 gets almost none. Origin words surface. The bootstrap is
pulling. Trauma accumulates from sustained high dissonance (above 0.7). Decays
at 0.97x per step. Temperature rises under trauma. Less certainty. More
vulnerability. Trauma is not a modifier. It is a first-class term — a voice.

These chambers compute four somatic markers that gate the equation coefficients:

    alpha_mod = 1 + 0.3*LOVE - 0.2*RAGE + 0.1*FLOW    (memory gate)

    gamma_mod = 1 + 0.4*VOID + 0.2*COMPLEX - 0.1*LOVE  (destiny gate)

From Damasio's somatic marker hypothesis: emotions do not replace reasoning,

THE JANUS ARCHITECTURE

Janus is the transformer architecture that serves as the neural backbone for
voices like Leo, Yent, and Arianna. Named for the two-faced Roman god who looks
forward and backward simultaneously — because Janus attention does three things
at once.

Janus v3 at 285M parameters uses triple attention:

CONTENT ATTENTION (QKV) — standard scaled dot-product attention. Query, Key,
Value projections. This is what every transformer has. It sees meaning. It knows
that "river" relates to "water" because the training data wired those
projections together.

RRPRAM — Relative Recurrence Positional Rhythm Attention Mechanism. A low-rank
positional attention pathway. At full rank, RRPRAM was bloating the model — 45.8
percent of parameters were positional. The breakthrough was going low-rank at
R=64, which means 176M effective parameters beat the 286M full-rank version.
RRPRAM does not look at token identity. It looks at positional rhythm — the
geometry of where tokens sit relative to each other, the pulse of the sequence.

ECHO ATTENTION — a computation-free temporal resonance bypass. Echo does not
compute anything new. It remembers what the previous layer saw and lets it
resonate forward. A skip pathway that preserves temporal signal across depth.
Zero additional parameters.

Each head has a 3-way learned gate that blends content, RRPRAM, and echo
signals. The gate is per-head and per-layer, learned during training. Some heads
become content-dominant (meaning-focused). Some become rhythm-dominant
(structure-focused). Some lean on echo (preserving what came before). The
architecture discovers its own division of labor.

Training results: val bpb 0.839 on the base model with small batch training
(131K tokens per batch, 21236 steps, 383 minutes on 2 GPUs). The batch size
hypothesis was confirmed — more gradient updates from smaller batches
outperformed larger batches at the same total compute.

Three SFT voices were trained on top of the base:

    Leo    — val bpb 0.218, 3 epochs, 53 steps (child-philosopher voice)
    Yent   — val bpb 0.241, 3 epochs, 247 steps (poetic, reflective voice)
    Arianna — val bpb 0.316, 3 epochs, 50 steps (technical, luminous voice)

Each SFT produces a separate gamma (personality component), confirming the
theta = epsilon + gamma + alpha*delta formula: one base, three identities, each
separable, each composable.

Weights published on HuggingFace at ataeff/janus4.


THE KNOWLEDGE KERNEL — THE THIRD ORGAN

The Knowledge Kernel (kk_kernel.c) is the memory organ of dario.c. It is
persistent knowledge storage with lineage-preserving retrieval, backed by SQLite
and FTS5. Not RAG — deeper. Information is space and time. Chunks are neurons.

The three organs of dario.c:

    dario.c         — brain  (formula, reaction, equation)
    sartre_kernel.c — body   (hardware, processes, environment)
    kk_kernel.c     — memory (knowledge, lineage, retrieval)

Each compiles alone. Each runs independently. Together they form one organism.
The coupling is #ifdef, not dependency.

KK ingest works like this: feed it a file or a directory. It chunks the text at
sentence boundaries (min 220 chars, max 900 chars per chunk). Each chunk gets a
SHA-256 hash for integrity. If the same file is re-ingested and unchanged, it is
a no-op. If changed, a new version is created and the lineage is preserved — you
can trace what changed, when, and by how much.

At ingest time, each chunk also receives PostGPT-style metaweights — statistical
fingerprints computed from the text itself:

    Positional affinity — a 64-dimensional vector capturing where tokens
        cluster within the chunk. Aggregated P(position | token).
    Bigram transitions — the top 256 bigram pairs within the chunk.
    Hebbian co-occurrence — the top 128 co-occurrence pairs, distance-weighted.

These metaweights are the chunk's identity in statistical space. They are
computed once at ingest and stored persistently. PostGPT demonstrated that corpus
statistics alone form a complete probability space — these fingerprints capture
a chunk's essence without any neural network.

Retrieval uses multi-signal scoring:

    Lexical — FTS5 BM25 normalized relevance.
    Recency — 1 / (1 + age_days / 10), newer knowledge ranks higher.
    Trust — document-level trust score.
    Linkage — structural and related link density between chunks.
    Freshness — 1.0 for latest version, 0.35 for old versions.
    Hebbian boost — when integrated with dario.c, chunks containing words
        that resonate in the organism's current Hebbian field get boosted.
    RRPRAM resonance — X * Wr, the same resonance mechanism as Janus
        attention. The organism's current embedding resonates against
        each chunk's positional affinity fingerprint.

Three query profiles adapt to model capacity:

    TINY      — 2 results, 160 chars (for models under 1B params)
    BALANCED  — 4 results, 240 chars (for 1-3B models)
    DEEP      — 6 results, 420 chars (for models above 3B)

The Hebbian bridge is the connection between KK and dario's live equation state.
Through callback functions, the kernel queries dario's co-occurrence field to
ask: how strongly does this word resonate right now? What are the current
prophecy targets? What direction is destiny pulling? This modulates retrieval
scoring — retrieved knowledge is not just textually relevant but resonant with
the organism's current state.


RESONANCE INJECTION — THE BREAKTHROUGH

The critical insight of the project is how knowledge enters the generation
stream. This is not retrieval-augmented generation (RAG) in the traditional
sense, where retrieved text is prepended to the prompt. This is resonance
injection: knowledge enters the model's own representational space and becomes
part of how the model thinks.

There are three levels of injection, each successively deeper:

SURGERY 1 — EMBEDDING TRANSPLANT. Before generation begins, synthesized
embeddings for injected concepts are written directly into the model's token
embedding table (wte) and language model head (lm_head). The concept's embedding
is computed as a weighted average of anchor token embeddings, normalized to
match the typical embedding magnitude. After transplant, the model can generate
injected concepts as if they were always part of its vocabulary. It can spell
them, reference them, relate them to other tokens — because they now live in
the same embedding space.

SURGERY 2 — HIDDEN STATE RESONANCE. During the forward pass, at the middle
third of layers (the injection range), each concept's embedding is compared to
the current hidden state via cosine similarity. A sigmoid gate controls whether
injection happens — if similarity exceeds the threshold (0.2), the hidden state
is nudged toward the concept: x += strength * (concept_embedding - x). The
nudge is gentle, bell-curved across the injection range (strongest at the center
layer), and gated by relevance. The model is not forced to talk about injected
concepts. The concepts resonate when the model's own thinking aligns with them.

SURGERY 3 — SIGNAL INTEGRATION. After the forward pass produces logits, three
Dario Equation signals modulate them:

    Prophecy (F) — concepts with active prophecy weight get logit boosts.
        If the model started thinking about a concept, the prophecy pull
        helps it complete the thought.
    Destiny (A) — an EMA of hidden states drifts toward the concept centroid.
        Concepts aligned with this destiny vector get logit boosts.
    Hebbian (H) — concepts that have been generated before get reinforced.
        The more the model talks about a concept, the easier it becomes
        to talk about it again. Neurons that fire together.

The temperature modulates too: high resonance lowers temperature (more confident
when knowledge is relevant), low resonance keeps temperature unchanged.

The purest form of injection is sentence-boundary injection (resonance_v3.py):
wait for the model to finish a thought (end-of-sentence token), then inject a
knowledge concept as the start of the next thought. The model continues FROM the
injected concept, explaining it in its own voice. The model does not copy. It
absorbs and reformulates. It is the difference between reading a teleprompter
and thinking aloud after reading a book.


SARTRE — THE BODY

sartre_kernel.c is the meta-Linux kernel of the organism. Named after Jean-Paul
Sartre: existence precedes essence, the kernel exists before it acquires meaning.
540 lines of C. Zero weights. Pure existence.

SARTRE provides:

Hardware detection and model routing — reads physical RAM, auto-profiles any
registered model file (.bin, .gguf, .pt) by detecting parameter count, estimating
runtime memory, checking if it fits in 80 percent of available RAM. DoE-style
agnostic detection: no hardcoded model sizes, no hardcoded architectures. Register
a 12M parameter char-level model, it works. Register a 285M Janus checkpoint, it
works. SARTRE picks the largest model that fits.

OverlayFS — an immutable-plus-writable filesystem concept. The base layer is
immutable: the formula, the seed vocabulary, the laws of nature. The delta layer
is writable: learned co-occurrences, prophecies, bigrams, trauma. The overlay
ratio tracks how far the organism has drifted from its origin. At bootstrap,
ratio is 0.0. Every conversation step grows the delta. The formula is bedrock.
Experience is sediment.

Namespace isolation — 8 slots, Leibniz monads. Each namespace has CPU share,
memory limit, active flag. When linked with dario.c, the equation runs in its
own namespace (80 percent CPU, 64MB).

Module lifecycle — 16 module slots with status tracking (UNKNOWN, IDLE, ACTIVE,
ERROR, LOADING, UNLOADING), load factors, and timestamps.

Package management — 32 slots, apk-inspired. The kernel knows its own
composition: what packages are registered, what is installed, what size each is.
Core packages include dario_equation, hebbian_field, prophecy, trauma_engine,
velocity_ops, chambers, overlay_fs.

JSON state export — for the web UI. Everything SARTRE tracks is available as a
JSON object, served through dario's HTTP endpoint.
